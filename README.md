# קורא ZIM — תוסף לאוצריא (מהדורת rework)

תוסף לקריאת קבצי **ZIM** (הפורמט של [Kiwix](https://www.kiwix.org) / [openZIM](https://openzim.org)) בתוך אוצריא — ויקיפדיה/ויקיטקסט offline, מילונים, אנציקלופדיות תורניות ועוד. מבוסס על ספריית **libzim** המהודרת ל-WebAssembly, הרצה בתוך Web Worker.

מהדורה זו היא **rework מקיף** של התוסף המקורי של [@האדם-החושב](https://github.com/zevisvei) — מציג ה-ZIM שהוא יצר לנו ברוב גאונותו. שיפרתי אותו מכמה מקומות, ואחרי 32 גרסאות (0.1.8 → 0.5.0) — הנה התוצאה.

## מה חדש במהדורה זו

- **א.** אפשרות לטעון כמה קבצי ZIM במקביל, והם נשמרים בזיכרון התוסף לפתיחה הבאה.
- **ב.** התוסף זוכר כעת את המיקום האחרון — הדף האחרון שהייתם בו — מה שלא היה קודם.
- **ג.** מהירות הטעינה של הקבצים עלתה משמעותית.
- **ד.** חיפוש נוח ומהיר בתוסף.
- **ה.** חיפוש חכם — כולל חיפוש במילים מנוקדות, חיפוש עם טעויות (כמו "כלומ" במקום "כלום"), וחיפוש עם הטיות: "שלום" ימצא גם "שהשלום"/"והשלום"/"כשהשלום" וכו'.
- **ו.** חלונית חיפוש בכותרות ובתוכן, עם הצגת התוצאות, הכותרות וההקשר שלהן.
- **ז.** חלונית ניווט בין הכותרות בדף שבו אתם נמצאים.
- **ח.** חיפוש בתוך הדף הפתוח, עם ניווט בין המופעים בדף.
- **ט.** חלונית להורדת המאגרים הבאים דרך התוסף:
  - **ויקימילון כלי קודש** — ויקימילון מסונן ומשוכתב שנוצר ע"י [@pcinfogmach](https://github.com/pcinfogmach) לפרויקט כלי קודש. תשואות חן לו על התוכן ועל העזרה.
  - **ויקיסוגיה** — מאתר ויקי.
  - **ויקישיבה** — מאתר ויקי.
- **י.** בעקבות הוראת הרבנים — התוסף כעת חוסם כל אפשרות של טעינת קובץ ZIM אחר שאינו ברשימה דלעיל, מכיוון שזהו תוסף לתוכנה תורנית, ולא ח"ו פתח לדברים לא טובים.

נשמח להערות והארות.

## הבעיות שנפתרו מהגרסה המקורית

המחבר המקורי ציין שלוש בעיות ידועות. כולן טופלו:

| # | הבעיה המקורית | הפתרון במהדורה זו |
|---|----------------|--------------------|
| 1 | **קבצים כבדים לא נטענים** | אסטרטגיית-טעינה לפי גודל: קבצים עד ~600MB נטענים **במלואם לזיכרון** (WORKERFS) לפתיחה מיידית ואמינה; קבצים גדולים יותר נטענים ב-**streaming עצל** (HTTP Range) בלי להוריד את כל הקובץ. בנוסף — טלאי-קריאה בגושים (bulk-read) שמחליף קריאה בית-אחר-בית. |
| 2 | **הממשק איטי** | עיצוב מחדש קליל, טעינת ערכים אסינכרונית, השהיית-הקלדה בחיפוש, שמירת-רצף (seq) שמונעת רינדור מיושן, ופתיחת ערכים מהירה. |
| 3 | **לא נשמרים מיקומים בין הפעלות** | ספריית-ארכיונים מתמשכת (persistent library), שמירת הקובץ האחרון, ושמירת **המיקום האחרון בכל ארכיון** (לפי token) — הכול נטען אוטומטית בהפעלה הבאה. |

## רשימת היכולות המלאה

**טעינה וארכיונים**
- טעינת קובץ `.zim` מקומי דרך File Picker (עם token מתמשך בין הפעלות).
- **ריבוי ארכיונים פתוחים בו-זמנית** ומעבר ביניהם (כמו Kiwix), עם בורר קבצים.
- זכירת הקובץ האחרון והמיקום האחרון בכל ארכיון.
- **זיהוי קבצים פגומים** לפני הטעינה — קובץ פגום/חלקי מקבל שגיאה ברורה מיד, במקום להיתקע בטעינה.

**חיפוש (חכם)**
- הצעות לפי כותרת (live) + חיפוש טקסט-מלא (Xapian) בתוך התוכן — עם מתג "☑ בתוכן".
- **התאמה עברית חכמה**: התעלמות מ**ניקוד וטעמים**, מ**אותיות סופיות** (ך=כ ם=מ ן=נ ף=פ ץ=צ), ומגרשיים/מרכאות; והתאמת **אותיות שימוש** (ה/ו/ב/כ/ל/מ/ש) — כך ש"ריבית" נמצא גם ב"הריבית".
- **תיקון אוט' של סיומת מילה**: הקלדת אות אמצעית בסוף מילה מתוקנת לצורתה הסופית (למשל "כלומ" → "כלום").
- **"האם התכוונת"** לראשי-תיבות: כשאין תוצאות, מנסה וריאנט עם גרשיים (למשל "רמבם" → "רמב״ם").
- **התאמת גבול-מילה** — לא מסמן חלקי-מילה שגויים.
- **קטע הקשר של 3 שורות** בכל תוצאה (לפני / התאמה / אחרי), שנבנה מהתוכן.

**קריאה וניווט**
- ניווט פנימי בין ערכים, כולל קישורים, תמונות וגיליונות-סגנון (מוטמעים כ-Blob URLs).
- הסרת JavaScript מכל ערך (אבטחה), ו**מיזוג רקע** של MediaWiki כדי למנוע פס אפור.
- **תוכן עניינים / ניווט לפי כותרות הדף**, מתעדכן לפי הערך הפתוח.
- **סימון צהוב** של מילת-החיפוש בדף ו**קפיצה** אליה, כולל **"חיפוש בדף"** עם ניווט הבא/קודם ומונה מופעים.

**ממשק**
- עיצוב מודרני (כפתורי-גלולה, תמיכת-מגע), פאנל-צד נשלף לתוצאות/ניווט, כותרת עם שם-הקובץ במרכז, וחלונית הורדות.

## איך זה עובד — פירוט מקצועי

### מנוע הקריאה: libzim (WebAssembly)
התוסף משתמש בספריית **libzim** של openZIM (אותה ספרייה שמפעילה את Kiwix), המהודרת ל-WebAssembly. היא רצה ב-**Web Worker** נפרד כדי לא לחסום את ה-UI, ומתקשרת עם הדף דרך `MessageChannel` (RPC).

### אתחול ב-WebView עם origin=null
בתוך WebView של `file://` (origin = null) אי-אפשר ליצור Worker מ-URL של אותו origin, וגם `fetch()` לקבצים מקומיים נכשל. הפתרון:
- קוד ה-worker נשלח כמחרוזת ב-`js/libzim-wasm-source.js` (‏`self.__libzimWasmSource`‏), וה-WASM כ-base64 ב-`js/libzim-wasm-data.js` (‏`self.__libzimWasmBase64`‏) — שניהם נטענים דרך `<script>` רגיל (שעובד מ-file://).
- נבנה **Blob URL** ל-worker, וה-WASM מוזרק ל-Emscripten דרך `Module.wasmBinary` (עם `locateFile` שמדלג על fetch של רשת).

### אסטרטגיית טעינה (הפתרון ל"קבצים כבדים")
- **≤ ~600MB** → הקובץ נמשך במלואו ל-Blob ומחובר דרך **WORKERFS** (‏`FileReaderSync`‏) — פתיחה מהירה ואמינה. פורמט ZIM ישן (v5) שפתיחתו מבצעת ריבוי קריאות קטנות נפתח כך ללא תקיעות.
- **גדול יותר** → **createLazyFile** עם בקשות **HTTP Range** סינכרוניות (streaming), בלי להעלות את כל הקובץ לזיכרון.

### טלאים ב-worker (מובנים ב-`libzim-wasm.js`, נעטפים בבנייה)
1. **RPC guard** — קריאה ל-action שלא קיים מחזירה `{__workerError}` במקום לקרוס.
2. **lazy URL + size** — העברת `url`/`size` ל-init לצורך createLazyFile.
3. **createLazyFile mount** — במצב עצל, במקום WORKERFS.
4. **דילוג על HEAD** ב-`cacheLength` כשהגודל ידוע (חוסך round-trip ומונע 404).
5. **קריאה בגושים** (bulk-read) בקריאה עצלה — במקום בית-אחר-בית, קריאה בגושים של 1MB.

### התאמת-חיפוש עברית (צד-לקוח)
מנוע החיפוש עצמו הוא libzim/Xapian (אין לו stemmer לעברית). מעליו נבנתה שכבת-התאמה: **קיפול** (folding) שמסיר ניקוד/טעמים ומאחד אותיות סופיות, והתאמת תת-מחרוזת עם גבולות-מילה ואותיות-שימוש — משמשת לסימון, לקטעי-ההקשר, ולתיקון השאילתה הנשלחת ל-libzim.

## מנגנון הגבלת קבצים (allowlist)

מהדורה זו כוללת **הגבלת-תוכן**: התוסף פותח רק קבצי ZIM שאושרו מראש. לכל קובץ מחושבת "טביעת-אצבע חלקית" — ‏`SHA-256`‏ על (‏1MiB ראשון + 1MiB אחרון + מחרוזת הגודל) — כך שהאימות מיידי גם לקבצים של ג'יגה-בייטים (נקראים רק ~2MB). הרשימה מאוחסנת בקוד כ-`SHA-256(salt + fingerprint)` (מעורפל).

> זוהי הגבלה "רכה" המיועדת למשתמשים רגילים; היא מונעת טעינה בשוגג של קבצים לא-מיועדים, אך אינה הגנה קריפטוגרפית מול תוקף נחוש. ניתן גם להתיר קובץ לפי מטא-דאטה `Name` שלו (עמיד לשינויי-גרסה).

## מאגרים להורדה

הקבצים מתארחים כ-**GitHub Release assets** (עוברים גם מאחורי סינון-תוכן, בניגוד ל-raw.githubusercontent). כפתורי ההורדה בתוסף פותחים אותם בדפדפן. ראו את [דף ה-Releases](https://github.com/YairDaniel11/otzaria-zim-plugin/releases) להורדה ישירה:

- **מילון כלי קודש** (`milon-klei-kodesh.zim`) — מילון ארמי-עברי ועוד, ~21,762 ערכים. מקור: [KleiKodesh](https://github.com/KleiKodesh).
- **ויקיסוגיה** (`wikisugya.zim`).

## בנייה

מתוך תיקיית הריפו, ב-Windows (PowerShell):

```powershell
.\build.ps1
```

הסקריפט עוטף מחדש את `js/libzim-wasm.js` + `js/libzim-wasm.wasm` ל-`libzim-wasm-source.js` + `libzim-wasm-data.js` (דרך `tools/wrap-assets.ps1`, נתיב יחסי), ואורז ל-`kiwix-zim-reader.otzplugin`.

## הרשאות

| הרשאה | למה |
|-------|-----|
| `ui.feedback` | הודעות/שגיאות למשתמש |
| `plugin.storage.read` / `write` | ספריית-ארכיונים, קובץ אחרון, מיקומים אחרונים, העדפות |
| `fs.user_files.read` | פתיחת קובץ ZIM שנבחר ב-File Picker וקריאתו ב-streaming |
| `events.subscribe:theme.changed` | התאמת ערכת-הנושא של אוצריא |

> התוסף **אינו** ניגש לרשת (‏`network.enabled=false`‏). ההורדות נפתחות בדפדפן החיצוני.
